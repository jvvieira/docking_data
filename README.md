# Docking Data Processing

This project processes molecular docking data from Rosetta scoring files (.sc files) and filters binding interactions based on statistical analysis.

## Project Overview

The pipeline:
1. Scans `.sc` (scoring) files from docking simulations for multiple protein complexes (DHFR, RAR, RXR, Ciclofilina)
2. Extracts interface scoring metrics (`I_sc` and `interface_delta`)
3. Applies statistical filtering (Shapiro-Wilk normality test, percentile-based cutoff)
4. Copies filtered PDB structures to an output directory

## Known Issues & Fixes

### FileNotFoundError: Ciclofilna Naming Inconsistency

**Error:** `FileNotFoundError: [Errno 2] No such file or directory: '../data/Ciclofilna/Ciclofilna_TS6_model1/...'`

**Root Cause:** The parent directory is named `Ciclofilina` (with an 'i'), but score file names are `Ciclofilna_*` (without the 'i'). When parsing the description field from .sc files, the extracted complex name matches the score file prefix, not the folder name.

**Solution:** In `process_data.py`, map the parsed complex name to the correct folder name:
```python
# Handle naming inconsistency: folder is 'Ciclofilina' but parsed as 'Ciclofilna'
folder_name = 'Ciclofilina' if complex1 == 'Ciclofilna' else complex1
source_path = f'../data/{folder_name}/{complex1}_{complex2}_{model}/...'
```

## Running the Script

```bash
python process_data.py
```

The script will:
1. Process all .sc files in the data directory
2. Print statistics about filtered cases
3. Copy matching PDB files to `./filtered_data/`
